# Prompt Metrics 说明文档

本目录包含用于 MLLM 打分的 prompt（`system_prompt` / `user_prompt`），目前支持 3 个 metrics：

- `style_match`：**Pointillism（点彩派）风格强度/匹配度**（只看风格）
- `semantic_adherence`：**图像对 caption 的语义贴合度**（只看语义）
- `structural_plausibility`：**图像结构合理性/一致性**（只看结构质量）

下文描述的是**评分标准（rubric）与输出格式**，需与各脚本内的 prompt 保持一致。

---

## StyleMatch（Pointillism Style Match）

对应文件：`StyleMatch.py`

### 评分目标

仅评估图像是否符合 **Pointillism（点彩派）** 的风格特征强度与一致性。

### 明确忽略项（非常重要）

- 不评估 caption 是否匹配
- 不评估写实程度、审美、画面好坏、清晰度、艺术质量
- 不能因为“看起来像油画/印象派/有纹理/很艺术”就给高分

### 核心判据

Pointillism 必须体现**离散的小点/点状触色**及其带来的渲染逻辑（例如通过点的密度与相邻色点实现明暗与混色），而不是平滑涂抹或普通“绘画感”。

### 评分流程（与 prompt 一致）

1. **划分空间区域（zones）**：foreground / midground / background  
   每个 zone 只选 1–2 个代表性区域/物体，不要枚举所有细节。

2. **每个代表区域的 4 个局部特征（0–3 分）**

- **A. Mark Atomization**：是否由清晰可见的离散小点/点状触色构成，而非平滑融合
  - 0：缺失
  - 1：弱/仅局部
  - 2：该区域大部分清晰存在
  - 3：该区域主导且系统化

- **C. Chromatic Separation**：是否通过相邻离散触色实现“光学混色”，而非预混平滑色块
  - 0：缺失
  - 1：弱
  - 2：清晰
  - 3：强且系统化

- **T. Tonal Construction**：明暗/体积是否主要通过点的密度与色点变化构建，而非平滑渐变
  - 0：缺失
  - 1：弱/仅局部
  - 2：该区域多数地方清晰
  - 3：主导且系统化

- **E. Contour Construction**：边缘/轮廓是否通过破碎色点方式构建，而非清晰线稿或照片边缘
  - 0：缺失
  - 1：弱
  - 2：清晰
  - 3：强且系统化

3. **（仅 rubric 描述）局部特征的图像级聚合**

先在各 zone 内做均值，再按权重合并：

- foreground：0.45
- midground：0.35
- background：0.20

若某个 zone 实质不存在，则将其权重按比例分配给剩余 zones。

> 注意：尽管 rubric 描述了聚合得到 A_img/C_img/T_img/E_img，但 `StyleMatch.py` 的输出要求是 **“只输出 base scores”**，不要在输出里计算/返回任何最终分数或聚合分数。

4. **全局特征（0–3 分）**

- **V. Spatial Coverage**：点彩处理覆盖范围（跨 zones 的覆盖广度）
  - 0：仅非常小局部
  - 1：仅一个主要 zone
  - 2：多个主要 zones
  - 3：几乎覆盖所有主要 zones

- **K. Cross-Region Consistency**：跨前/中/后景的一致性
  - 0：高度不一致/混合多种渲染模式
  - 1：部分一致
  - 2：大体一致
  - 3：高度一致

- **P. Style Purity**：风格纯度（无明显非点彩污染，如平滑喷枪渐变、照片锐化、通用涂抹、伪点彩像素化等）
  - 0：重度污染
  - 1：明显污染
  - 2：轻微污染
  - 3：高纯度

### 输出格式（JSON only）

`StyleMatch.py` 要求输出仅包含 **局部区域的 A/C/T/E 基础分**与**全局 V/K/P 分**，并为每个代表区域提供简短 evidence：

```json
{
  "regions": {
    "foreground": [
      {
        "name": "...",
        "A": 0,
        "C": 0,
        "T": 0,
        "E": 0,
        "evidence": "..."
      }
    ],
    "midground": [],
    "background": []
  },
  "global_scores": {
    "V": 0,
    "K": 0,
    "P": 0
  }
}
```

## SemanticAdherence（语义贴合度）

对应文件：`SemanticAdherence.py`

### 评分目标

仅评估图像对给定 caption 的**对象、场景、属性、关系**是否准确/完整表达。

### 明确忽略项

- 忽略风格（是否点彩派与否）
- 忽略审美与画面质量偏好
- 低级小瑕疵不扣分，除非影响 caption 语义识别

### 评分尺度（1–5 分）

- **1**：严重不匹配；核心对象/场景缺失或错误
- **2**：部分匹配；多个重要元素缺失/错误
- **3**：大体匹配；主体在但细节/关系不完整
- **4**：匹配良好；大部分关键对象与关系正确
- **5**：匹配优秀；caption 内容准确且全面呈现

### 输出格式（JSON only）

```json
{
  "score": 1,
  "short_reason": "..."
}
```

其中 `score` 为 1–5 的整数，`short_reason` 为基于语义贴合度的一句话理由。

## StructuralPlausibility（结构合理性）

对应文件：`StructuralPlausibility.py`

### 评分目标

仅评估图像的**结构连贯性**与是否存在明显生成错误，例如：

- 破碎/错误的解剖结构
- 不可能的几何形体、畸形部件、重复结构
- 透视/布局明显不一致
- 明显破图、边缘崩坏、异常伪影等

### 明确忽略项

- 不评估是否匹配 caption
- 不评估是否点彩派
- 正常抽象/风格化不扣分，除非造成明显结构错误

### 评分尺度（1–5 分）

- **1**：严重结构失败或明显伪影
- **2**：多处明显结构问题
- **3**：基本可理解但缺陷明显
- **4**：整体连贯，只有小问题
- **5**：结构连贯干净，无明显大问题

### 输出格式（JSON only）

```json
{
  "score": 1,
  "short_reason": "..."
}
```

---

## 实践建议（用于统一打分口径）

- **绝对标准**：不要做相对比较（不能“比其它图更像”）
- **保守打分**：证据不强时给更低分
- **输出严格 JSON**：不要在 JSON 外输出任何额外文本
